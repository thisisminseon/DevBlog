Troubleshooting

課題と解決

開発中に直面した技術的課題と、その原因分析・解決過程です。

1投稿アップロードの失敗
問題
記事作成時に「投稿に失敗しました」エラーが発生し、データが保存されない。
原因
Eclipse の -parameters コンパイラオプション未設定により、@RequestParam のパラメータ名が解決できなかった。
解決
全ての @RequestParam と @PathVariable にアノテーション引数を明示的に記述。例：@RequestParam("title")
2記事編集ページの500エラー
問題
記事の編集ページにアクセスすると500 Internal Server Errorが発生。
原因
@PathVariable("id") のアノテーション引数が未指定で、パラメータバインドに失敗。
解決
@PathVariable("id") と明示的に記述し、Controllerの全メソッドを同様に修正。
3Flexmarkライブラリの解決失敗
問題
「The import com.vladsch cannot be resolved」エラーでアプリケーションが起動不可。
原因
EclipseのGradle依存関係の同期が切れ、外部ライブラリが認識されなくなった。
解決
Gradle → Refresh Gradle Project → Project → Clean → IDE再起動で依存関係を再同期。
4ファビコンが表示されない
問題
staticフォルダにファビコンを配置しても、ブラウザに反映されない（5回の試行）。
原因
Eclipse(STS)がファイルシステムの変更を bin/main に自動同期しない仕様。Cleanするとコピーしたファイルが消される。
解決
STS Package Explorerから直接ファイルをインポートし、Thymeleafの th:href で参照。
5画像プレビュー削除ボタンの無反応
問題
記事作成時、画像プレビューのホバー削除ボタンがクリックできない。
原因
親要素の overflow: hidden が削除ボタンを隠し、z-index の設定不足でクリックイベントが届かなかった。
解決
overflow: hidden を削除、z-index: 10 を追加、DataTransfer APIでFileListを動的管理。
